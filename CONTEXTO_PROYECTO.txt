# INTI WEAR - Proyecto de E-commerce Streetwear Andino

## Resumen del Proyecto

Tienda online de streetwear con diseños inspired in Andean culture from Cusco, Peru. Built with Next.js 14, TypeScript, Tailwind CSS, Prisma (SQLite), Zustand, NextAuth, and Stripe.

---

## Stack Original

- Next.js 14 (App Router)
- TypeScript
- Tailwind CSS
- Prisma + SQLite (dev.db)
- Zustand (carrito con persistencia)
- Lucide React (icons)  

---

## Modificaciones y Agregados Completos

### 1. BASE DE DATOS (Prisma)

**Schema actualizado en `prisma/schema.prisma`:**

```prisma
model Producto {
  id          String   @id @default(cuid())
  nombre      String
  slug        String   @unique
  descripcion String
  precio      Float
  categoria   String
  imagenUrl   String
  tallas      String
  colores     String
  stock       Int      @default(0)
  activo      Boolean  @default(true)
  createdAt   DateTime @default(now())
}

model User {
  id            String    @id @default(cuid())
  name          String?
  email         String    @unique
  emailVerified DateTime?
  image         String?
  password      String?
  createdAt     DateTime  @default(now())
  updatedAt     DateTime  @updatedAt
  orders        Order[]
}

model Order {
  id              String   @id @default(cuid())
  userId          String?
  user            User?    @relation(fields: [userId], references: [id])
  items           String   // JSON serializado
  subtotal        Float
  shipping        Float
  total           Float
  nombre          String
  email           String
  telefono        String
  direccion       String
  ciudad          String
  estado          String   @default("pendiente")
  paymentStatus   String   @default("pending")
  paymentId       String?
  createdAt       DateTime @default(now())
  updatedAt       DateTime @updatedAt
}

model Reserva {
  id        String   @id @default(cuid())
  productos String   // JSON serializado
  total     Float
  nombre    String
  whatsapp  String
  estado    String   @default("pendiente")
  createdAt DateTime @default(now())
}
```

**Seed con 8 productos:**

### 2. API ROUTES CREADAS

| Endpoint | Método | Descripción |
|----------|--------|-------------|
| `/api/productos` | GET | Lista productos (filtro `?categoria=xxx`) |
| `/api/productos/[slug]` | GET | Producto individual por slug |
| `/api/reservas` | POST | Guarda reservas WhatsApp |
| `/api/auth/[...nextauth]` | GET/POST | Autenticación NextAuth |
| `/api/auth/register` | POST | Registro de usuarios |
| `/api/checkout` | POST | Crea sesión de Stripe |
| `/api/orders` | GET/POST | CRUD de órdenes |

### 3. AUTENTICACIÓN (NextAuth v4)

**Archivos creados:**
- `src/lib/auth.ts` - Configuración de NextAuth conCredentials
- `src/app/api/auth/[...nextauth]/route.ts` - Handler de autenticación
- `src/app/api/auth/register/route.ts` - Endpoint de registro
- `src/types/next-auth.d.ts` - Extensión de tipos

**Características:**
- Login con email/password
- Registro de usuarios
- Sesiones JWT
- Protección de rutas (cuenta)
- Integración con Prisma

### 12. NOTAS PARA DEPLOY

- Necesitas agregar tus propias keys de Stripe en `.env`
- Para WhatsApp, reemplaza `51XXXXXXXXX` con tu número real
- La DB es SQLite local (dev.db) - para producción usar Vercel Postgres
- El proyecto está configurado para Next.js 14
